  一、全局 / GLOBAL

[GLOBAL-01] [CRITICAL] request.ts 直接将后端错误消息 Toast 给用户
  文件: frontend/src/utils/request.ts
  问题: 所有 API 失败时统一弹 Toast 显示 data.message。Spring Boot 报 404 时
        返回 "No static resource api/homepage/feed." 这类技术信息被直接展示给用户。
        没有 silent 选项，无法对非关键 API 静默失败。
  修复: request 函数签名改为 options & { silent?: boolean }，两处 showToast 加
        if (!options.silent) 判断。getHomeContentApi 加 silent:true。
  状态: [x]

[GLOBAL-02] [HIGH] WeChat/Guest 登录后 userId 是字符串，导致大量 API 调用被跳过
  文件: frontend/src/pages/login/index.vue
  问题: wxLogin 写死 userId = 'wx_guest_001'，guestLogin 写死 userId = 'guest_temp_001'。
        history.vue / start.vue / user/index.vue 等所有页面都有校验：
          const numericId = Number(userId);
          if (!isNaN(numericId) && numericId > 0) { ... }
        字符串 userId 转换后为 NaN，导致所有后端数据请求全部跳过，页面空白或显示假数据。
  状态: [x] (Guest 模式不再写入假字符串 userId，改为清除 userId/token)

[LOGIN-01] [CRITICAL] 注册成功后未保存 token，后续所有鉴权请求 401
  文件: frontend/src/pages/login/index.vue, 第 132-140 行
  问题: registerApi 返回 User 对象（无 token），只存了 userId 和 userInfo。
        request.ts 依赖 getStorageSync('token') 拼 Authorization 头，
        注册后首次进入 app 所有需要鉴权的接口全部 401。
        正确做法：注册成功后自动调用 loginApi 获取 token，或让后端注册接口也返回 token。
  修复: registerApi 成功后紧接着调用 loginApi，将返回的 token/userId/userInfo
        一并存入 storage，流程与正常登录完全一致。
  状态: [x]
