<五大Tab栏与首页核心功能>
1. 首页 (Home)：核心功能的入口聚合。顶部是全局搜索和用户头像，下方有一个问候语行（Hello，XXX），再下面是四个核心AI工具的图标入口（职业测评、技能地图、AI简历诊断、模拟面试），往下是横向滚动的"职场加油站"资讯卡片流，再往下是"热门话题"排行榜列表。点头像可以弹出ActionSheet跳到个人主页或退出登录。
2. 消息 (Messages)：所有通知的集散地。顶部大标题"消息"，下面是一个iOS原生风格的Segment分段控制器，分成"沟通"（模拟HR对话消息列表）、"通知"（系统和AI任务完成提醒）、"投递"（求职状态更新，带进度标签如"复筛中"、"已投递"、"已通过"）三个tab。消息卡片带头像、未读标记、时间戳等。
3. 助手 (Assistant)：24小时在线的AI职业顾问。整个页面是一个标准的聊天界面——顶部固定导航栏显示AI头像和在线状态，中间是消息流（支持欢迎卡片+快捷提问标签、时间分隔线、用户/AI气泡、打字动画），底部固定输入栏带发送按钮。
4. 简历库 (Resume)：简历资产管理中心。顶部大标题+副标题，蓝色渐变Hero卡片显示简历数量，下方是简历卡片列表（带PDF图标、文件名、状态标签、预览按钮和更多操作），底部有一个虚线边框的"新建简历"入口按钮，点击弹出自定义的底部ActionSheet（模板填写 / 上传PDF / 取消），支持chooseMessageFile选PDF上传。
5. 我的 (User)：个人中心和各种设置。顶部蓝色渐变卡片显示头像、昵称、学校信息（已登录态）或"未登录"引导登录按钮（未登录态）。下面是一个统计条（测评/模拟面试/简历数量），然后是两组菜单列表——"我的资产"（简历库、收藏的文章、浏览记录）和"外观与辅助"（深色模式Switch开关、字号选择药丸：紧凑/标准/大号）。最底下是退出登录按钮。点击头像可以chooseImage更换头像。

<四大核心AI工具（从首页进入的二级页面）>
1. 职业测评 (assessment)：进入后有返回按钮+大标题+Step1提示条，蓝色渐变状态卡片显示测评进度，下面列出两个测评卡片（MBTI 93题 / 霍兰德 120题），每个卡片有图标、标题、描述、标签和"开始"按钮。点击进入quiz答题页，完成后跳转result结果页展示AI画像。
2. 技能地图 (map)：顶部返回+"技能地图"大标题+其他按钮。深色头部卡片显示当前岗位（可点击切换前端/后端/产品/数据分析）和技能掌握进度环45%。主体是一个纵向时间轴，从L1到L4四个阶段节点——已完成的节点(✅绿色)、进行中的节点(🔥蓝色脉冲动画+进度条)、未解锁的节点(🔒半透明)。点击任一节点弹出底部Modal显示核心考点和学习建议。

<多端系统架构（A、B、C三端）>
为了保证系统的专业性与数据隔离，整个软硬件系统严格按照 A、B、C 三端来划分架构逻辑：
- C端（客户端）：面向大学生。也就是学生注册登录后使用的前台系统，包含上述的首页（含四大AI工具）、消息、助手、简历库、我的这五个核心功能模块，主要提供具体的求职辅助与规划服务。
- B端（机构端）：面向学校的辅导员或就业指导中心老师。老师登录后，系统会展示本范围（如某学院）内学生脱敏后的就业意向数据大盘、面试能力短板统计等。它是一个数据辅助决策后台，用来帮助学校更精准地开展就业指导工作。
- A端（总控端）：面向开发与运营团队。这是系统的最高权限总后台，主要用来为各个高校或学院（B端）分配和开通账号、维护系统底层数据（如动态更新"职业地图"里的行业与核心技能节点）、配置大模型的 API 以及监控系统的整体运行状态。

产品标准使用顺序:
用户首次打开应用，会先进入登录/注册页（支持账号密码登录、微信快捷登录、游客体验三种方式）。登录后落到"首页"，从金刚区四个图标开始使用核心功能：先做"职业测评"认识自己，再看"技能地图"了解目标岗位的能力要求，然后去底部Tab"简历库"里创建简历并用"AI简历诊断"针对性打磨，最后进"模拟面试"实战练习。过程中有问题随时在"助手"Tab里跟AI对话。系统通知和HR沟通都在"消息"Tab里汇总。

【核心技术栈与视觉约束】
- 前端 (C端框架)：基于 uni-app + Vue 3 (组合式 API + TypeScript) + Vite 构建，面向 H5 与微信小程序。
- 前端 (UI与样式)：没用第三方组件库，全部手写原生CSS，走的是iOS/Apple HIG风格——圆角卡片、毛玻璃效果、SF Pro系统字体栈。
- 前端 (视觉主题)：主色调科技蓝(#2563eb系)，辅助色活力橙(用在面试按钮呼吸动画)。支持深色模式(.is-dark类名切换)。五个Tab页风格高度统一：浅灰底(#f5f5f7) + 白色卡片 + 蓝色渐变强调。
- 前端 (TabBar)：五个tab——首页、消息、助手、简历库、我的。自定义tabbar图标。
- 后端 (微服务)：采用 Java 17 + Spring Boot 3.x 开发 RESTful API。
- 持久化：MySQL 存核心结构化数据，MongoDB 存测评/画像等灵活文档，Redis 负责鉴权缓存。
- 对象存储：Aliyun OSS 负责简历与视频流存放。
- API对接状态：登录/注册接口已跑通(@/api/user)，个人中心页已调用getUserInfoApi读取用户信息。其余模块目前前端用mock数据和setTimeout模拟异步。
