Sprint 3 敏捷看板任务拆分与估算表

总计 Story Points: 50

User Story 1：账户体系与个人中心
所属模块：登录/注册 + 个人中心 + 全局设置
Story Points: 8
描述: 作为新用户，我希望通过多种方式（账号密码/微信/游客体验）快速进入应用，登录后能管理个人信息和偏好设置，以便我的数据能跨设备同步且有安全保障。

任务拆分:
Task 1.1 [前端 - 成员A] (Story Points: 3): 开发登录/注册页（含Segment Tab切换登录/注册、表单验证、微信快捷登录、游客体验入口、用户协议勾选），以及"我的"Tab个人中心页面（已登录态/未登录态两套UI、头像上传、深色模式切换、字号选择、退出登录）。
Task 1.2 [前端 - 成员B] (Story Points: 2): 搭建 B端/A端 Web 后台基础骨架与路由守卫（基础 Admin 模板套壳）。
Task 1.3 [后端 - 成员C] (Story Points: 2): 重构 users 表结构，实现基于 JWT 的多端 (A/B/C Role) 角色鉴权登录接口（含注册和登录两个接口）。
Task 1.4 [联调 - 成员D] (Story Points: 1): 前后端联调登录注册流程，验证Pinia/Storage全局用户状态同步，确保未登录时拦截到登录页。

接收标准 (Acceptance Criteria):
场景 1: 首次进入应用
假设 用户没登录就打开应用
当 页面加载完成
那么 自动跳到登录/注册页
并且 页面顶部显示蓝色渐变背景和"开启你的求职闭环"标题
并且 底部有Segment Tab切换登录和注册两种模式

场景 2: 多种登录方式
假设 用户在登录页面
当 用户选择不同登录方式
那么 账号密码方式需要填写手机号和密码，点击登录后调用loginApi
并且 微信快捷登录点击后显示"微信授权中"loading
并且 游客体验方式直接写入本地storage一个临时账号进入首页
并且 所有方式都需要先勾选用户协议才能提交

场景 3: 个人中心已登录态
假设 用户已登录并进入"我的"Tab
当 页面加载完成
那么 顶部蓝色渐变卡片显示头像、昵称和学校信息
并且 中间统计条显示测评(2)、模拟面试(5)、简历(3)的数量
并且 下方菜单分两组："我的资产"（简历库、收藏的文章、浏览记录）和"外观与辅助"（深色模式Switch、字号三档选择）
并且 底部有红色"退出登录"按钮

场景 4: 深色模式与字号切换
假设 用户在"我的"Tab的"外观与辅助"区域
当 打开深色模式Switch
那么 当前页面切换为暗色主题背景(#0f172a)
当 点击字号药丸切换为"大号"
那么 保存设置到本地Storage，显示"已保存"轻提示


User Story 2：职业认知测评与技能地图
所属模块：职业测评(assessment) + 技能地图(map)
Story Points: 13
描述: 作为还没想清楚未来方向的大学生，我想通过专业量表测试看看自己的性格和职业倾向，然后在一个可视化的技能树上看清楚目标岗位从入门到专家各阶段需要什么能力，让我知道该往哪补。

任务拆分:
Task 2.1 [前端 - 成员B] (Story Points: 3): 开发测评主页（蓝色渐变头部状态卡+精选测评列表，含MBTI和霍兰德两个测评卡片），以及quiz答题页和result结果页。
Task 2.2 [前端 - 成员C] (Story Points: 3): 开发"技能地图"页面的纵向时间轴组件（L1-L4四个阶段节点，区分已完成/进行中/未解锁状态，含脉冲节点动画和底部Sheet详情弹窗）。
Task 2.3 [后端 - 成员D] (Story Points: 3): 设计 MongoDB 测评文档结构，开发测评判分与大模型画像生成接口。
Task 2.4 [后端 - 成员A] (Story Points: 2): 设计 MySQL 职业节点表，开发技能地图的数据拉取接口。
Task 2.5 [联调 - 成员B] (Story Points: 2): 联调测评答题链路与地图节点的动态渲染（根据测评结果推荐岗位和路径）。

接收标准 (Acceptance Criteria):
场景 1: 打开职业测评页
假设 用户从首页金刚区点击"职业测评"进入
当 页面加载完成
那么 顶部有"返回"按钮和"职业测评"大标题
并且 显示Step 1提示条说明测评是第一步
并且 蓝色渐变卡片显示"已完成0项 / 总计2项专业测评"的进度
并且 下方列出MBTI(93题/5分钟)和霍兰德(120题/8分钟)两个可点击的测评卡片

场景 2: 查看技能地图
假设 用户从首页点击"技能地图"进入
当 页面加载完成
那么 深色头部卡片显示当前岗位名称（默认"前端开发工程师"）和进度环(45%)
并且 岗位名称可以点击切换（前端/后端/产品/数据分析四个选项）
并且 主体显示L1→L4四个纵向时间轴节点
并且 L1(HTML/CSS)显示绿色已完成✅，L2(JS/TS)显示蓝色进行中🔥带进度条(60%)，L3/L4显示灰色未解锁🔒（半透明）

场景 3: 查看技能详情
假设 用户在技能地图页面
当 点击某个技能节点（如L2 JS/TS）
那么 从底部弹出一个Sheet面板
并且 面板显示技能名称、掌握度(60%)、核心考点列表（ES6+、Event Loop、Promise、TS泛型）和学习建议


User Story 3：简历管理与AI诊断
所属模块：简历库Tab + AI简历诊断
Story Points: 8
描述: 作为要找工作的学生，我想集中管理我的简历文件，还能让AI根据我想投的岗位JD帮我诊断简历的匹配度、指出不足并一键优化，省得我自己瞎改。

任务拆分:
Task 3.1 [前端 - 成员C] (Story Points: 3): 开发"简历库"Tab页（蓝色渐变Hero卡片+简历列表+新建简历入口+自定义ActionSheet+PDF上传解析+重命名/分享/删除操作）。
Task 3.2 [前端 - 成员D] (Story Points: 2): 开发"AI简历诊断"页面（简历选择+JD粘贴+带进度条的全屏加载动画+诊断结果卡含匹配度分数环和优劣势分析+一键生成按钮）。
Task 3.3 [后端 - 成员A] (Story Points: 2): 开发简历数据的 CRUD 接口及 Aliyun OSS 附件/头像上传逻辑。
Task 3.4 [联调 - 成员B] (Story Points: 1): 编写 Prompt 模板对接大模型，联调简历定向润色接口。

接收标准 (Acceptance Criteria):
场景 1: 简历库主页
假设 用户进入"简历库"Tab
当 页面加载完成
那么 顶部显示"简历库"大标题和"管理你的求职资产"副标题
并且 蓝色渐变Hero卡片显示当前简历总数
并且 下方显示已有的简历卡片列表（每张含PDF图标、文件名、更新日期、状态标签、预览按钮和更多操作按钮）
并且 底部有虚线边框的"新建简历"按钮

场景 2: 上传简历
假设 用户点击"新建简历"按钮
那么 从底部弹出自定义ActionSheet（非系统原生）
并且 Sheet包含"模板填写"（跳到模板表单页）、"上传PDF简历"（打开文件选择器限定PDF）和"取消"三个选项
当 选择PDF上传并选中文件
那么 显示"上传并解析中"loading动画
并且 完成后新简历出现在列表顶部，显示"解析成功"标签

场景 3: 简历操作
假设 用户在简历列表中点击某张简历的"···"按钮
那么 弹出系统ActionSheet显示"重命名"、"分享"、"删除"三个选项
当 选择删除
那么 弹出确认Modal，确认后从列表移除该简历

场景 4: AI诊断全流程
假设 用户从首页进入"AI简历诊断"页
当 选择了简历并粘贴了JD文本后点击"开始AI诊断"
那么 全屏覆盖加载动画出现，进度条从0到100分5个阶段推进（准备→提取特征→解析结构→比对JD→生成方案）
当 动画完成
那么 显示诊断结果卡：分数环显示85分匹配度，列出优势项和待优化项
并且 底部提供"一键生成特化版简历"按钮


User Story 4：沉浸式模拟面试与面后复盘
所属模块：模拟面试(interview)
Story Points: 13
描述: 作为容易在面试中紧张卡壳的学生，我想跟AI面试官做一次真实感拉满的模拟面试——开摄像头、选岗位、选难度，面完了能看到各维度的打分和具体的改进方向，这样我真上场的时候心里就有底了。

Task 4.1 [前端 - 成员D] (Story Points: 3): 开发面试大厅页（暗色状态卡+面试设置表单+权限请求弹窗+呼吸动画开始按钮）和面试房间沉浸式UI（接入WebRTC/摄像头）。
Task 4.2 [前端 - 成员A] (Story Points: 2): 开发面后复盘报告页（综合分数圆环+四维度条形图+AI建议卡片+毛玻璃底部操作栏）。
Task 4.3 [后端 - 成员B] (Story Points: 3): 对接大模型与流式响应，实现面试官的动态追问与打分逻辑。
Task 4.4 [后端 - 成员C] (Story Points: 3): 开发面试记录云端存储逻辑与复盘报告生成接口。
Task 4.5 [联调 - 成员D] (Story Points: 2): 联调音视频流权限与 AI 实时对话链路。

接收标准 (Acceptance Criteria):
场景 1: 面试前准备
假设 用户从首页进入"模拟面试"
当 页面加载完成
那么 暗色头部卡片显示"AI面试官"标题和历史最高分(82)
并且 下方表单可选择目标岗位(4个选项)、面试难度(3个等级)、关联简历
并且 蓝色提示框说明需要麦克风和摄像头权限
并且 底部固定毛玻璃工具栏有带蓝色呼吸脉冲动画的"开启模拟面试"按钮

场景 2: 权限请求与进入面试
假设 用户选好岗位后点击"开启模拟面试"
当 未选择岗位
那么 显示"请先选择目标岗位"轻提示
当 已选择岗位
那么 先弹Modal说明需要麦克风和摄像头权限
并且 用户确认后系统依次请求麦克风(scope.record)和摄像头(scope.camera)权限
并且 权限通过后跳转到面试房间页面

场景 3: 面试复盘报告
假设 面试结束跳转到report页面
当 页面加载完成
那么 顶部显示"面试复盘报告"标题和岗位信息、面试耗时
并且 综合得分圆环（绿色渐变）显示85分和"表现优秀"评价
并且 维度拆解区域用条形图展示四个维度分数（专业90/表达82/逻辑85/抗压75）
并且 AI面试官建议区域列出具体的改进建议
并且 底部固定毛玻璃工具栏有"返回面试大厅"按钮


User Story 5：首页入口与AI助手
所属模块：首页(home) + 助手(assistant)
Story Points: 8
描述: 作为用户，我需要一个清晰的首页能快速找到各核心功能入口和行业资讯，并在任何时候都能跟AI助手聊天，获得基于我个人画像的定制化求职建议。

任务拆分:
Task 5.1 [前端 - 成员A] (Story Points: 2): 开发首页（搜索栏+头像+问候语+四大金刚区图标入口+横滚资讯卡片流+热门话题排行榜），并配置pages.json路由和五大TabBar。
Task 5.2 [前端 - 成员B] (Story Points: 3): 开发"助手"Tab的聊天界面（固定导航栏+欢迎卡片+快捷提问标签+消息气泡+打字跳动动画+固定输入栏+发送按钮状态变色）。
Task 5.3 [后端 - 成员C] (Story Points: 2): 封装全局大模型流式对话接口 (SSE/WebSocket)，处理上下文记忆。
Task 5.4 [联调 - 成员D] (Story Points: 1): 联调助手对话接口，确保前端打字机输出体验顺畅。

接收标准 (Acceptance Criteria):
场景 1: 首页布局
假设 用户登录后进入首页
当 页面加载完成
那么 顶部显示搜索栏（圆角白色）和用户头像（点击弹出ActionSheet可跳转个人中心或退出登录）
并且 下方问候语显示"Hello，XXX"（XXX为用户昵称或默认"体验用户"）
并且 四个金刚区图标按钮分别是职业测评(🧭蓝底)、技能地图(🗺紫底)、AI简历诊断(✨粉紫底)、模拟面试(🎤橙底)
并且 "职场加油站"区域是可横向滚动的彩色渐变封面资讯卡片
并且 "热门话题"区域是带排名编号的列表（前3名蓝色高亮）

场景 2: AI助手对话
假设 用户进入"助手"Tab
当 页面加载完成
那么 顶部导航栏固定显示AI头像(🤖蓝色渐变圆)、名称"AI职业顾问"和绿色在线状态
并且 聊天区显示欢迎卡片（含三个快捷提问标签：优化简历/面试技巧/行业薪资）
并且 已有一条AI的初始问候消息

场景 3: 发送消息与回复
假设 用户在助手页面底部输入框输入文字
当 输入框有内容时
那么 发送按钮从灰色变为蓝色(#2563eb)
当 点击发送
那么 用户消息以蓝色气泡显示在右侧
并且 0.5秒后AI回复区域显示三个跳动圆点的打字动画
并且 1.5秒后打字动画替换为AI的文字回复（白色气泡在左侧）
